Intro to Algorithms

THE HOOK

Have you ever made a peanut butter and jelly sandwich? Or followed a recipe? Or given someone directions? If so, you've already used an algorithm!

WHAT IS AN ALGORITHM?

In Computer Science, an algorithm is a precise, step-by-step procedure for solving a problem or accomplishing a task. It's like a recipe for a computer: a list of instructions that, if followed exactly, always produce the correct result.

Why Do We Need Algorithms?
  • Computers can't improvise or guess‚Äîthey need every step spelled out clearly.
  • Algorithms allow us to automate tasks, solve problems efficiently, and build everything from simple calculators to complex apps like Google Maps or TikTok.
  • Good algorithms can make programs faster, use less memory, and even save money or lives (think: air traffic control, medical devices, or self-driving cars).

REAL-WORLD ANALOGY

Imagine teaching a robot to tie its shoes. You can't just say "tie your shoes"‚Äîyou must break it down into tiny, exact steps: grab the laces, cross them, pull tight, make a loop, etc. If you skip a step or are unclear, the robot will get stuck or make a mistake. This is why precision and order matter in algorithms.

Key Properties of Algorithms
  • Input: What goes in (like numbers to add, or a list to sort).
  • Output: The result (like the sum, or the sorted list).
  • Definiteness: Each step is clear and unambiguous.
  • Finiteness: The process always ends after a finite number of steps.
  • Effectiveness: Each step is simple enough to actually do.

EXAMPLES

  • Recipe: An algorithm for making cookies.
  • Directions: An algorithm for getting from your house to school.
  • Sorting: An algorithm for putting books in order by title.
  • Searching: An algorithm for finding a word in a dictionary.
Sample Algorithm (Pseudocode):
function findMax(numbers) {
  max = numbers[0];
  for each number in numbers {
    if (number > max) {
      max = number;
    }
  }
  return max;
}

This algorithm finds the largest number in a list. Notice how every step is clear and nothing is left to chance!

WHY STUDY ALGORITHMS?

  • They are the foundation of all computer programs.
  • They help us solve problems efficiently and creatively.
  • Understanding algorithms makes you a better problem solver in life, not just in code!

Divide & Conquer

THE HOOK

Imagine you have a giant phone book and need to find one name. Would you start at page 1 and check every name? Or would you flip to the middle, see if the name is before or after, and cut the book in half each time? That's the magic of Divide & Conquer!

WHAT IS DIVIDE & CONQUER?

Divide & Conquer is a powerful algorithmic strategy where you break a big problem into smaller, easier pieces, solve each piece, and then combine the answers. It's like solving a puzzle by working on small sections, then putting them together.

Why Use Divide & Conquer?
  • It makes hard problems easier by breaking them down.
  • It can make algorithms much faster (think: searching, sorting, even multiplying huge numbers!).
  • It's the secret sauce behind famous algorithms like Binary Search, Merge Sort, and more.

REAL-WORLD ANALOGY

Suppose you need to find a word in a dictionary. Instead of starting at "A" and flipping one page at a time, you open to the middle. If your word comes before that page, you only look in the first half. If after, you look in the second half. Each time, you cut the problem in half‚Äîsuper efficient!

Key Steps of Divide & Conquer
  • Divide: Split the problem into smaller subproblems.
  • Conquer: Solve each subproblem (sometimes using Divide & Conquer again!).
  • Combine: Put the solutions together to solve the original problem.

EXAMPLE: BINARY SEARCH

Suppose you have a sorted list of numbers and want to find if a number is in the list. Instead of checking every number, you:

Sample Algorithm (Pseudocode):
function binarySearch(list, target) {
  if list is empty:
    return false
  middle = middle index of list
  if list[middle] == target:
    return true
  else if target < list[middle]:
    search left half
  else:
    search right half
}

This method keeps cutting the list in half until it finds the answer or runs out of options. Super fast!

WHY STUDY DIVIDE & CONQUER?

  • It helps you solve big problems by thinking smaller.
  • It leads to some of the fastest algorithms in computer science.
  • It's a strategy you can use in life: break big tasks into smaller steps!

Data Structures

THE HOOK

Ever tried to organize your backpack, your closet, or even your phone apps? The way you arrange things can make life much easier‚Äîor much harder! That's the power of data structures.

WHAT ARE DATA STRUCTURES?

Data structures are special ways to organize and store information so computers can use it efficiently. Just like you use shelves, drawers, or folders to keep things tidy, computers use data structures to keep data organized.

Why Do We Need Data Structures?
  • They help computers find, add, or remove information quickly.
  • Choosing the right structure can make programs much faster and use less memory.
  • They're the building blocks for everything from video games to search engines!

REAL-WORLD ANALOGY

Imagine a cafeteria: students line up for food (like a Queue‚Äîfirst in, first out), while trays are stacked up (like a Stack‚Äîlast in, first out). Each way of organizing things has its own rules and best uses!

Common Data Structures
  • Array: Like a row of lockers, each with a number.
  • Stack: Like a stack of pancakes‚Äîtake the top one first (Last In, First Out).
  • Queue: Like a line at the movies‚Äîfirst in, first out.
  • Linked List: Like a treasure hunt, where each clue leads to the next.
  • Tree: Like a family tree, branching out from one ancestor.
  • Hash Table: Like a super-fast address book for looking things up.

EXAMPLES

  • Stack: Undo history in a drawing app (last action undone first).
  • Queue: Print jobs waiting for the printer.
  • Array: High scores in a game, stored in order.
Sample Stack Algorithm (Pseudocode):
stack = []
stack.push("plate1")
stack.push("plate2")
top = stack.pop() // removes "plate2"

Stacks let you add and remove items from the top only‚Äîjust like a stack of trays!

WHY STUDY DATA STRUCTURES?

  • They make your programs faster and smarter.
  • They're used everywhere in computer science.
  • Understanding them helps you solve real-world problems, from organizing music playlists to building apps!

Searching & Sorting

THE HOOK

Ever tried to find your favorite shirt in a messy closet or put your books in order? Computers face the same challenges‚Äîfinding and organizing information quickly!

WHAT ARE SEARCHING & SORTING?

Searching means looking for something in a collection (like finding a contact in your phone). Sorting means putting things in order (like arranging books by title or number).

Why Do We Need Them?
  • Searching helps us find information fast‚Äîthink Google or your phone's search bar.
  • Sorting makes searching even faster and keeps things organized.
  • Many apps and websites use these algorithms behind the scenes every day!

REAL-WORLD ANALOGY

Imagine a library: if the books are in random order, finding one takes forever. If they're sorted by title or author, you can find any book in seconds. That's the magic of sorting and searching!

Common Searching Algorithms
  • Linear Search: Check each item one by one (like looking through a messy pile).
  • Binary Search: Quickly find an item in a sorted list by cutting the search space in half each time.
Sample Linear Search (Pseudocode):
function linearSearch(list, target) {
  for each item in list {
    if (item == target) {
      return true
    }
  }
  return false
}

Linear search is simple but can be slow for big lists.

Common Sorting Algorithms
  • Bubble Sort: Compare and swap adjacent items until everything is in order (like bubbling up the biggest number).
  • Merge Sort: Divide the list, sort each part, then merge them together (uses Divide & Conquer!).
Sample Bubble Sort (Pseudocode):
function bubbleSort(list) {
  repeat until no swaps {
    for i from 0 to length-2 {
      if (list[i] > list[i+1]) {
        swap list[i] and list[i+1]
      }
    }
  } }

Bubble sort is easy to understand, but there are faster ways for big lists!

WHY STUDY SEARCHING & SORTING?

  • They help computers (and you!) find and organize information quickly.
  • They're used in almost every app, website, and device.
  • Understanding them makes you a better problem solver and coder.
